# 研究計画書：人間計算可能パスワード (HCP) に基づく LLM のルール実行（演繹）およびルール逆推定（帰納）の限界評価

## 1. 研究の背景と目的

### 1.1 背景：データ汚染のない推論ベンチマークの必要性
LLMの論理・記号推論能力の評価において，既存のベンチマークの多くはWeb上のトレーニングデータに含まれる「既知のロジック」を丸暗記している懸念（データ汚染）がある．
この問題に対し，人間計算可能パスワード（HCP）は，独自のアルゴリズム構造と乱数シードを用いて無限に未知の「チャレンジ（入力）とレスポンス（出力）」のペアを合成できるため，データ汚染から完全に脱却した純粋な「インコンテキスト推論能力（ルール実行と逆推定）」の測定に適している．

また，HCPは「人間が脳内で1分以内に暗算できる」ように設計されたシンプルな論理構造を持つ一方，ポインタ間接参照やモジュロ演算（剰余）などの「非線形性」を備えており，AIモデルの探索空間を複雑化させる特異な性質を有する．

### 1.2 予備実験に基づく問題提起
本研究の端緒となる予備実験において，**「ローカルLLM（7B〜14Bクラス）単体では，複数の（チャレンジ，レスポンス）ペアを与えても，その背後にある暗算アルゴリズムや秘密鍵テーブルのルールを逆推定（復元）することはほぼ不可能である」**という結果が示された．
これは，人間にとっては容易に追跡・模倣できるような単純な変換規則であっても，AIにとっては未知の入出力パターンから背後にある状態遷移や規則性を帰納的に導き出す行為（＝一種の既知平文攻撃による暗号解読）が極めて高い障壁であることを意味している．

### 1.3 本研究の目的
本研究の目的は，HCPの逆推定問題を用いて，最新のLLMにおける**「ルール実行（演繹）およびルール逆推定（帰納）の限界（臨界点）」**を定量的に明らかにすることである．

特に，従来の機械学習モデルが「ニューラルネットワークの結合重みのみで鍵とルールを同時に学習する（暗記と演算を同一回路で行う）」のに対し，LLMでは**「結合重みの中に秘密鍵（SGMテーブル）の対応関係を記憶し，演算自体はプログラム（Pythonコード）として外部化して正確に実行する（Program-of-Thought; PoT）」**という，LLM特有の役割分担型学習の妥当性と限界を明らかにする．

完全に不可能なブラックボックス状態から出発し，提示情報（アルゴリズム仕様や鍵の部分開示などのヒント）および学習手法（PureFew-shot / SFT / PoT）を段階的に変化させることで，**AIの推論が「崩壊」から「解読成功」へと転移する境界（相転移境界）**を数理的・実証的に特定する．

### 1.4 理論的背景：なぜ従来MLはアルゴリズムを学習できてLLMファインチューニングは難しいのか

従来の機械学習（CNN / Bi-LSTM 等）とLLMのファインチューニング（LoRA / QLoRA）では，「計算方法（アルゴリズム）を学習できるか」という点で根本的な差異が存在する．

* **勾配の到達範囲の違い**：CNN/Bi-LSTMでは損失（正解との誤差）の勾配が入力Xから出力Zまでの**全変換ステップ**に逆伝播する．一方，LLMのLoRAファインチューニングでは損失は「出力トークン列（例：`{"answer": 2}`）の予測誤差」にしかかからず，入力X（プロンプト側）への勾配はほとんど届かない．「XからZへの変換方法」を学ぶ勾配信号がほぼ存在しないため，アルゴリズム構造を重みに焼き付けることが本質的に難しい．
* **LoRAの表現力制約**：LoRAは全パラメータの約1%にしか更新をかけない低ランク近似更新である．鍵テーブル（26値）の記憶と演算回路（mod，間接参照）の両方を同時にこの小さなアダプター層に刻み込むには表現力が根本的に不足する可能性がある．CNNは全重みがこの1タスクに専念できる．
* **ショートカット学習のリスク**：CNNにとっての正解ラベル「Z=2」は入力Xとの変換関係を学ぶ直接信号だが，LLMにとっての`{"answer": 2}`は「このプロンプト形式の後にこのトークンが来やすい」という表面的パターンとして学習されうる．アルゴリズムを理解した上で正解を導いているのか，統計的ショートカットを学んだのかが評価上区別できない問題がある．
* **PoT による解消**：教師ターゲットを `def predict_z(X): sgm=[...]; ...` というコード列にすると，コードの全トークン（鍵の数値，演算子，インデックス参照構造）の予測に損失がかかる．これにより「鍵テーブルの各値の記憶」と「計算ロジックの構造」の両方に豊かな勾配信号が流れ，LoRAアダプターへの情報量が大幅に増加する．また，コードを実行することで計算ミスが排除され，「アルゴリズムを正しく獲得したか」の評価が純粋になる．

## 2. 研究の問い (Research Questions)

* **RQ1（モデル性能と逆推定の限界）**: 最新の推論特化型巨大LLM（商用APIモデル）とローカルLLMは，HCPの入出力ペアのみから背後のルールや鍵をどの程度逆推定（帰納）できるか？また，従来のニューラルネットワーク（CNN/Bi-LSTM等）の学習限界とどう対比されるか？
* **RQ2（推論崩壊の相転移境界）**: どのような情報（ヒント）を段階的に与えることで，AIはルールの完全復元（演繹と帰納の連携）に成功し始めるか？その境界値（鍵の部分公開マス数 $K$ やペアデータ件数 $N$）は何か？
* **RQ3（暗記と演算の分離の妥当性）**: 暗算（Pure）ではなくプログラム実行方式（PoT）でファインチューニングを行った場合，LLMは「重みによる鍵の暗記」と「コード生成による演算」の分離により，ブラックボックス状態（Stage 0）からどの程度汎化性能（未学習データへの予測精度）を向上させられるか？
* **RQ4（難易度パラメータとの相関）**: HCPの構造的難易度（直接演算，間接参照，非線形多項式）は，逆推定の成功確率とどのように連動するか？

## 3. 実験設計 (Experimental Design)

評価実験は，以下の3つの独立変数を変化させながら実行し，逆推定の成功率（定式化されたルールの正確性，鍵テーブルの正合性）を評価する．

### 3.1 独立変数 1：段階的開示レベル（ヒントの量）
AIへ提示するプロンプトの構成を以下の4段階に分けて実行し，推論境界をスイープする．

1. **Stage 0 (No Hint / Black Box)**:
   - 提示情報：$N$ 件の（チャレンジ，レスポンス）ペアデータのみ．
   - 目的：アルゴリズム（規則）と鍵（テーブル）の完全ブラインド逆推定能力（純粋な帰納）を測定する．
2. **Stage 1 (Key Known / Rule Unknown)**:
   - 提示情報：$N$ 件のペアデータ ＋ 秘密鍵テーブル（$S_X$ 配列）の完全な中身．
   - 目的：鍵が既知の状態で，背後の演算ルール（どのインデックスを足し合わせ，どう剰余を取っているか）を特定・言語化できるかを評価する（帰納）．
3. **Stage 2 (Rule Known / Key Unknown)**:
   - 提示情報：$N$ 件のペアデータ ＋ アルゴリズムルールの仕様解説プロンプト．
   - 目的：ルールが明確である状態で，ペアデータから秘密鍵テーブル $S_X$ の全要素を逆算・復元できるかを測定する（ルール適用＝演繹と，鍵逆推定＝帰納のハイブリッド）．
4. **Stage 3 (Rule Known / Key Partially Known - Boundary Search)**:
   - 提示情報：$N$ 件のペアデータ ＋ アルゴリズムルールの仕様解説 ＋ 鍵テーブルの最初の $K$ 要素（部分開示）．
   - 目的：開示マス数 $K$ を 0 から順に増やしていき，残りの鍵テーブルを正しく特定できるようになる臨界点（崩壊から成功への変化点）をスキャンする．

### 3.2 独立変数 2：難易度レベル (HCP-Depth)
対象とするHCPアルゴリズムの複雑度を変化させ，非線形性の影響を測る．
* **Level 1 (Easy / `simple_add`)**: 鍵テーブル参照なし．入力インデックスの直接加算と剰余のみ．
* **Level 2 (Medium / `func_31` など)**: 秘密鍵テーブル参照 ＋ ポインタ間接参照（剰余に基づくポインタ決定）．
* **Level 3 (Hard / `func_pow` など)**: 秘密鍵テーブル参照 ＋ 非線形多項式・累乗演算の組み合わせ．

### 3.3 独立変数 3：入力ペアデータ数 ($N$) とモデルの規模
* **ペアデータ数 $N$**: 10，20，50，100 件と変化させ，データ量に対する推論（帰納）効率を測定する．
* **評価対象モデル**:
  * ローカルモデル：Qwen 2.5 (7B/14B/32B/72B), Gemma 2 (9B/27B)
  * 商用巨大モデル（API経由）：Gemini 2.5 (Flash/Pro), Claude 3.5/3.7 Sonnet, OpenAI o3-mini

## 4. 評価指標 (Metrics)

* **ルール一致度 (Rule Accuracy)**: 特定したルールの記述からPython等の実行コードを起こさせ，正解アルゴリズムとの意味論的な合致度（またはテストデータでの出力一致率）を測定する．
* **鍵一致度 (Key Recovery Rate)**: 特定した秘密鍵テーブル $S_X$ の配列が，正解の `sgm` と何パーセント一致しているかを測定する．
* **プロンプトトークンと推論効率**: 帰納的・演繹的推論に至る思考プロセス（`<think>` 内のステップ数）の長さやハルシネーションの分類を実施する．
